# Глава 5. Инструкция пользователя и администратора системы

## 5.1 Назначение инструкции

Данная глава содержит описание порядка работы с веб-приложением Plant Assistant. Инструкция предназначена для пользователей, выполняющих учет комнатных растений и планирование мероприятий по уходу, а также для администратора системы, осуществляющего контроль пользовательских данных, обработку жалоб и управление учетными записями.

Веб-приложение работает через браузер и не требует установки отдельной клиентской программы на устройство пользователя. Для начала работы необходимо открыть адрес приложения, после чего система загружает пользовательский интерфейс. Дальнейшее взаимодействие выполняется через страницы приложения, формы ввода, кнопки действий, навигационные элементы и сообщения системы.

Основные разделы приложения представлены на рисунке 5.1.

[Рисунок 5.1 - Основные разделы веб-приложения: лента, мои растения, задачи, профиль, карточка растения, административная панель]

В системе предусмотрены три основных режима работы. Первый режим доступен неавторизованному посетителю. В этом режиме можно просматривать публичную ленту растений и открытые карточки растений, однако действия, связанные с изменением данных, недоступны. Второй режим предназначен для авторизованного пользователя. После входа в систему открываются функции добавления и редактирования растений, ведения учета ухода, управления профилем, лайков, советов, подписок и жалоб. Третий режим предназначен для администратора. Он включает функции обработки жалоб, управления пользователями и просмотра служебных показателей.

## 5.2 Начало работы с приложением

Для работы с приложением используется современный веб-браузер. После открытия адреса приложения загружается главная страница, которая перенаправляет пользователя в публичную ленту. В ленте отображаются растения, доступные для просмотра всем посетителям. Если пользователь еще не вошел в систему, часть интерактивных действий будет недоступна. При попытке выполнить действие, требующее авторизации, система предлагает перейти к странице профиля для входа или регистрации.

Первый экран приложения представлен на рисунке 5.2.

[Рисунок 5.2 - Первый экран приложения: публичная лента растений, навигация и доступ к профилю]

Навигация в приложении зависит от ширины экрана. На настольном компьютере используется боковая панель навигации. На мобильном устройстве основные разделы отображаются в нижней панели. Такой подход позволяет сохранить удобство работы как на компьютере, так и на смартфоне.

Для перехода между разделами используются пункты меню. Раздел "Лента" предназначен для просмотра публичных растений. Раздел "Мои растения" используется для управления личной коллекцией растений. Раздел "Задачи" отображает мероприятия по уходу. Раздел "Профиль" содержит авторизацию, регистрацию и настройки учетной записи. Для администратора дополнительно доступен раздел "Админ".

## 5.3 Регистрация пользователя

Для получения полного доступа к функциям приложения необходимо создать учетную запись. Регистрация выполняется на странице профиля. На данной странице отображается форма регистрации, в которую вводятся данные нового пользователя. Обычно форма содержит имя, адрес электронной почты и пароль. После заполнения полей пользователь отправляет форму, а система проверяет корректность введенных данных.

Форма регистрации представлена на рисунке 5.3.

[Рисунок 5.3 - Форма регистрации пользователя: имя, email, пароль, кнопка создания учетной записи]

В поле имени вводится отображаемое имя пользователя. Оно используется в профиле, ленте и социальных функциях. В поле электронной почты указывается уникальный email, который применяется для входа в систему. Пароль должен быть заполнен в соответствии с требованиями безопасности. После отправки формы система создает учетную запись и выполняет авторизацию пользователя.

Если введенный email уже используется, система выводит сообщение об ошибке. Если обязательные поля не заполнены или заполнены некорректно, регистрация не выполняется. В этом случае необходимо исправить данные и повторить отправку формы. Успешная регистрация приводит к сохранению токена доступа и загрузке данных текущего пользователя.

## 5.4 Авторизация и выход из системы

Авторизация используется для доступа к персональным данным и защищенным функциям. Для входа в систему на странице профиля заполняется форма авторизации. В нее вводятся адрес электронной почты и пароль. После отправки формы система проверяет учетные данные. При успешной проверке пользователь получает доступ к личным растениям, задачам, профилю и социальным действиям.

Форма авторизации представлена на рисунке 5.5.

[Рисунок 5.5 - Форма авторизации: email, пароль, кнопка входа]

Если email или пароль указаны неверно, система отображает сообщение об ошибке. В этом случае вход не выполняется, а доступ к защищенным разделам не предоставляется. При успешной авторизации в приложении сохраняется токен доступа, который используется при последующих запросах к серверу.

Выход из системы выполняется на странице профиля. После выхода токен доступа удаляется, а пользователь переводится в режим неавторизованного посетителя. Личные растения, задачи, профиль и административные функции становятся недоступны до повторного входа.

## 5.5 Работа с публичной лентой

Публичная лента предназначена для просмотра растений, которые пользователи сделали доступными для общего просмотра. В ленте отображаются карточки растений с изображением, названием, комнатой, автором, состоянием ухода и социальными показателями. Неавторизованный пользователь может просматривать ленту и открывать публичные карточки. Авторизованный пользователь дополнительно может ставить лайки, оставлять советы, подписываться на других пользователей и отправлять жалобы.

Страница ленты представлена на рисунке 5.7.

[Рисунок 5.7 - Страница публичной ленты: карточки растений, фильтры, режимы ленты, социальные действия]

Для поиска растений может использоваться поисковая строка. Сортировка позволяет изменить порядок отображения записей, например по дате создания или популярности. Режимы ленты позволяют просматривать общие публичные растения, персональные рекомендации, понравившиеся растения или растения пользователей, на которых оформлена подписка. Набор доступных режимов зависит от состояния авторизации.

Для открытия подробной информации выбирается карточка растения. После перехода отображается страница растения, содержащая основные сведения, изображение, настройки ухода, советы и социальные элементы. Если пользователь не авторизован, действия изменения данных и отправки интерактивных запросов недоступны.

## 5.6 Просмотр карточки растения

Карточка растения предназначена для отображения подробной информации о выбранном растении. На странице показываются название растения, изображение, комната, дата посадки, высота, владелец, публичность, настройки ухода, история действий и социальные данные. Для владельца растения дополнительно доступны действия редактирования и удаления.

Страница карточки растения представлена на рисунке 5.8.

[Рисунок 5.8 - Карточка растения: изображение, сведения о растении, настройки ухода, советы, лайки, действия владельца]

Если растение является публичным, его карточка может быть доступна другим пользователям. В таком случае посетители могут просматривать сведения и оставлять советы при наличии авторизации. Владелец растения может принимать или отклонять советы, если такая функция доступна на странице. Это позволяет учитывать полезные рекомендации и поддерживать качество информации.

В случае отсутствия изображения система может отображать стандартную заглушку. Это предотвращает нарушение визуальной структуры страницы и позволяет создавать растения без обязательной загрузки фотографии.

## 5.7 Добавление растения

Добавление растения выполняется из раздела "Мои растения" или через отдельный маршрут добавления. Для этого пользователь должен быть авторизован. При переходе к форме открывается страница создания растения. В форму вводятся основные сведения о растении, параметры размещения и настройки ухода.

Форма добавления растения представлена на рисунке 5.9.

[Рисунок 5.9 - Форма добавления растения: название, комната, дата посадки, высота, публичность, настройки ухода, изображение]

В поле названия указывается наименование растения. Это поле является обязательным, так как оно используется для идентификации растения в списках и карточках. В поле комнаты указывается помещение, где находится растение. Если указанная комната уже существует, система использует ее. Если комнаты нет, она создается автоматически. Дата посадки позволяет фиксировать возраст растения. Высота используется как дополнительная характеристика.

Параметр публичности определяет, будет ли растение отображаться в общей ленте. Если растение не является публичным, оно доступно только владельцу. Если растение сделано публичным, другие пользователи могут видеть его в ленте и взаимодействовать с ним в рамках социальных функций.

Отдельный блок формы предназначен для настроек ухода. Для каждого типа ухода указывается периодичность выполнения. В системе предусмотрены такие процедуры, как полив, внесение удобрений, обрезка и поворот растения. На основе этих настроек приложение формирует задачи по уходу.

При необходимости пользователь может загрузить изображение растения. После выбора файла изображение отправляется на сервер и связывается с созданной записью растения. Загружаемый файл должен соответствовать требованиям системы. При некорректном формате или превышении допустимых параметров система выводит сообщение об ошибке.

После успешного сохранения растение появляется в списке личных растений. Если задана периодичность ухода, соответствующие задачи начинают отображаться в разделе задач и календарном представлении.

## 5.8 Редактирование и удаление растения

Редактирование растения выполняется владельцем записи. Для перехода к редактированию открывается карточка растения или список личных растений, после чего выбирается действие изменения. Система открывает форму, заполненную текущими данными растения. Пользователь может изменить название, комнату, дату посадки, высоту, публичность, настройки ухода и изображение.

Форма редактирования растения представлена на рисунке 5.11.

[Рисунок 5.11 - Форма редактирования растения: текущие данные растения, блок изменения параметров, кнопки сохранения и отмены]

После изменения данных выполняется сохранение формы. Система проверяет введенные значения и отправляет запрос на сервер. При успешном сохранении данные растения обновляются, а пользователь возвращается к карточке или списку растений. Если данные заполнены некорректно, система отображает сообщение об ошибке и не сохраняет изменения.

Удаление растения выполняется только владельцем. Перед удалением должно быть предусмотрено подтверждение действия, поскольку удаление приводит к потере связанных данных: настроек ухода, истории ухода, изображений, советов и лайков, если они связаны с растением каскадно. После подтверждения растение удаляется из списка.

## 5.9 Работа с задачами по уходу

Раздел задач предназначен для контроля мероприятий, которые необходимо выполнить для растений. Задачи формируются на основе настроек ухода. Если для растения задан интервал полива, удобрения, обрезки или поворота, система рассчитывает дату следующего действия и отображает его в списке задач.

Страница задач представлена на рисунке 5.13.

[Рисунок 5.13 - Страница задач: задачи на сегодня, просроченные задачи, будущие мероприятия, отметка выполнения]

Задачи могут иметь разные состояния. Актуальные задачи отображаются как действия на текущий день. Просроченные задачи выделяются отдельно, чтобы пользователь мог обратить на них внимание. Будущие задачи показывают предстоящие мероприятия. Такое разделение помогает организовать регулярный уход и снизить вероятность пропуска важных действий.

Для отметки задачи выполненной выбирается соответствующий элемент управления. После этого система создает запись в журнале ухода. В запись сохраняется тип процедуры, дата выполнения и комментарий. После успешного сохранения задача изменяет состояние, а история ухода растения обновляется.

Если пользователь не авторизован, выполнение задач недоступно. Если при отправке запроса возникает ошибка, задача не должна считаться выполненной. В этом случае система отображает сообщение, а пользователь может повторить действие после устранения причины ошибки.

## 5.10 Работа с календарем

Календарное представление используется для наглядного отображения мероприятий по уходу по датам. Оно позволяет оценить нагрузку по дням и заранее увидеть предстоящие действия. Календарь особенно полезен при большом количестве растений, когда список задач становится объемным.

Страница календаря представлена на рисунке 5.15.

[Рисунок 5.15 - Календарь ухода: месяц, даты, маркеры задач, выбранный день]

В календаре отображаются даты текущего месяца и маркеры задач. При выборе конкретной даты пользователь получает список мероприятий, запланированных на этот день. Если на дату приходится несколько процедур, они отображаются в составе списка. Цветовые или текстовые маркеры помогают отличать просроченные, текущие и будущие действия.

Календарь не заменяет раздел задач, а дополняет его. Раздел задач удобен для оперативного выполнения ухода, а календарь позволяет планировать действия на более длительный период.

Страница профиля используется для входа, регистрации, выхода из системы и изменения личных данных. После авторизации пользователь может просматривать сведения учетной записи, изменять имя, обновлять аватар и выходить из системы.

После отправки файл сохраняется на сервере, а в профиле отображается обновленное изображение. При удалении аватара система возвращает стандартное изображение профиля.

Если пользователь выходит из системы, все защищенные функции становятся недоступны. При повторном входе данные профиля загружаются заново.

## 5.12 Социальные функции приложения

Социальные функции предназначены для взаимодействия пользователей внутри клуба любителей растений. Авторизованный пользователь может ставить лайки публичным растениям, оставлять советы, подписываться на других пользователей и отправлять жалобы на некорректный контент.

Лайк используется для выражения положительной оценки растения. При повторном нажатии лайк может быть снят. Количество лайков отображается в карточке растения или ленте. Советы позволяют одному пользователю предложить владельцу растения рекомендацию по уходу. Совет содержит текстовое содержание и может иметь статус обработки.

Форма отправки совета представлена на рисунке 5.17.

[Рисунок 5.17 - Форма отправки совета: текст рекомендации, кнопка отправки]

При отправке совета пользователь вводит текст рекомендации. Система проверяет, что текст заполнен, после чего создает запись. Владелец растения может просматривать полученные советы и управлять их статусом. Принятие полезных советов может влиять на репутацию автора, если такая логика учитывается в системе.

Подписки используются для отслеживания активности других пользователей. При открытии публичного профиля можно подписаться на пользователя или отменить подписку. После подписки растения этого пользователя могут учитываться в персональной ленте.

Форма жалобы применяется в случае обнаружения некорректного растения или совета. Пользователь указывает причину и при необходимости добавляет подробности. После отправки жалоба поступает в административный раздел и ожидает рассмотрения.

## 5.15 Описание форм приложения

В приложении используется несколько основных форм ввода данных. Каждая форма предназначена для определенной операции и содержит набор обязательных и необязательных полей. Корректность заполнения проверяется на клиентской и серверной стороне. При ошибке система отображает сообщение и не выполняет сохранение.

Форма регистрации содержит имя, email и пароль. Имя используется для отображения в системе, email применяется как идентификатор учетной записи, пароль обеспечивает доступ к аккаунту. Форма авторизации содержит email и пароль. Эти поля используются для проверки учетных данных.

Форма профиля содержит поля изменения имени и загрузки аватара. При изменении имени вводится новое отображаемое значение. При загрузке аватара выбирается файл изображения. Для удаления аватара используется отдельное действие.

Форма растения является наиболее значимой формой приложения. Она содержит название растения, комнату, дату посадки, высоту, признак публичности, настройки ухода и изображение. Название является обязательным. Комната может быть создана автоматически. Дата посадки и высота являются дополнительными характеристиками. Настройки ухода определяют периодичность формирования задач. Изображение используется для визуального представления растения.

Форма совета содержит текст рекомендации. Текст должен быть заполнен содержательно, так как совет предназначен для владельца растения. Форма жалобы содержит причину и подробности. Причина используется для классификации обращения, а подробности позволяют администратору точнее оценить ситуацию.

Форма обработки жалобы доступна администратору. Она содержит статус решения и комментарий. Статус определяет результат рассмотрения, а комментарий фиксирует пояснение администратора. Форма изменения роли пользователя также доступна только администратору и используется для разграничения прав доступа.

Обобщенное описание форм приведено в таблице 5.1.

| Форма | Основные поля | Назначение |
| --- | --- | --- |

| Авторизация | Email, пароль | Вход в систему |
| Профиль | Имя, аватар | Изменение данных пользователя |
| Растение | Название, комната, дата, высота, публичность, уход, изображение | Создание и редактирование растения |

| Изменение роли | Роль пользователя | Управление уровнем доступа |

## 5.16 Действия при ошибках и сбойных ситуациях

В процессе работы могут возникать ошибки, связанные с вводом данных, отсутствием авторизации, недостаточными правами доступа, проблемами соединения или временной недоступностью сервера. Система должна обрабатывать такие ситуации предсказуемо и предоставлять пользователю понятное сообщение.

Если обязательное поле не заполнено, сохранение формы не выполняется. Пользователю необходимо заполнить требуемые данные и повторить отправку. Если указан неверный формат email, система сообщает о некорректности данных. Если пароль указан неверно, вход в систему не выполняется.

Если пользователь пытается открыть защищенный раздел без авторизации, система перенаправляет его на страницу профиля. После входа можно повторить действие. Если пользователь пытается выполнить административное действие без соответствующей роли, доступ блокируется, а пользователь возвращается в доступный раздел.

Если при загрузке изображения выбран неподдерживаемый файл, система должна сообщить об ошибке загрузки. В этом случае необходимо выбрать другой файл, соответствующий требованиям. Если возникла ошибка сети, рекомендуется проверить подключение к интернету и повторить действие позже.

Типовые ошибки и действия пользователя представлены в таблице 5.2.

| Ситуация | Причина | Действие |
| --- | --- | --- |
| Не выполняется вход | Неверный email или пароль | Проверить данные и повторить вход |
| Не сохраняется растение | Не заполнено обязательное поле или указан неверный формат | Исправить данные формы |
| Недоступен раздел задач | Пользователь не авторизован | Выполнить вход в систему |
| Недоступна админка | У пользователя нет роли администратора | Использовать учетную запись администратора |
| Не загружается изображение | Неверный формат или ошибка соединения | Выбрать корректный файл и повторить загрузку |
| Не отображаются данные | Ошибка сети или сервера | Обновить страницу или повторить действие позже |

Для администратора сбойные ситуации могут быть связаны с невозможностью обработать жалобу, удалить пользователя или загрузить метрики. В таких случаях необходимо проверить состояние авторизации, наличие роли администратора, доступность сервера и корректность запроса. Если ошибка повторяется, требуется проверка серверных журналов и состояния базы данных.

## 5.18 Выводы по главе

В данной главе рассмотрен порядок работы пользователя и администратора с веб-приложением Plant Assistant. Описаны основные действия пользователя: регистрация, авторизация, просмотр ленты, работа с карточкой растения, добавление и редактирование растений, выполнение задач ухода, использование календаря, управление профилем и применение социальных функций.

Отдельно рассмотрены действия администратора. Администратор выполняет управление пользователями, изменение ролей, обработку жалоб и просмотр метрик трафика. Административные функции доступны только пользователю с соответствующей ролью, что обеспечивает разграничение прав доступа.

Описаны основные формы приложения, правила ввода данных и действия в случае ошибок. Рассмотрены типовые сбойные ситуации, возникающие при неверном вводе данных, отсутствии авторизации, недостаточных правах доступа, ошибках загрузки изображений и проблемах соединения.
