# Глава 2. Анализ существующих аналогов и прототипов

В рамках разработки веб-приложения клуба был проведён анализ существующих решений аналогичной направленности. Целью анализа является выявление функциональных возможностей современных веб-сервисов, а также оценка степени их применимости в рассматриваемой предметной области.

При рассмотрении аналогов особое внимание уделялось удобству работы с данными о растениях, возможностям планирования ухода, а также доступности интерфейса для широкой аудитории пользователей.

## 2.1 Анализ веб-сервиса AssistPlant
Сервис AssistPlant представляет собой веб-инструмент, ориентированный на планирование ухода за растениями с использованием календарной модели. Основное внимание уделяется формированию расписания регулярных действий.

**Функциональные возможности:**
- планирование мероприятий по уходу;
- формирование расписания полива и других процедур;
- система напоминаний;
- ведение истории действий;
- интеграция с календарями.
**Преимущества:**
- простая и понятная логика работы;
- наглядное представление задач в виде календаря;
- возможность регулярного контроля выполнения мероприятий.

**Недостатки:**
- отсутствие русскоязычного интерфейса может затруднять использование для части пользователей;
- ограниченные возможности структурирования данных снижают удобство при работе с большим количеством растений;
- слабая визуализация информации уменьшает наглядность представления данных.

В целом сервис ориентирован на базовое планирование и подходит преимущественно для простых сценариев использования.
## 2.2 Анализ веб-версии сервиса PlantIn
PlantIn представляет собой платформу, предоставляющую рекомендации по уходу за растениями и справочную информацию. Веб-версия обеспечивает доступ к основным возможностям системы.
**Функциональные возможности:**
- рекомендации по уходу;
- напоминания о поливе и удобрении;
- база знаний;
- идентификация растений;
- учет состояния.
**Преимущества:**
- наличие русскоязычного интерфейса;
- развитая справочная составляющая;
- удобное представление информации.
**Недостатки:**
- ограниченность веб-версии;
- основной акцент на рекомендациях, а не на систематизации данных;
- часть функций доступна только при оформлении подписки.
Таким образом, сервис может рассматриваться как вспомогательный инструмент, но не как полноценная система учета.

## 2.3 Анализ веб-приложения Gardenize
Сервис Gardenize предназначен для ведения журнала ухода за растениями и фиксации наблюдений.
**Функциональные возможности:**
- ведение журнала;
- учет выполненных действий;
- хранение фотографий;
- добавление заметок;
- структурирование данных.
**Преимущества:**
- высокая степень детализации;
- возможность накопления полной истории наблюдений;
- удобная работа с большими объемами информации.
**Недостатки:**
- отсутствие русскоязычного интерфейса;
- сравнительно сложный интерфейс;
- ограниченные возможности автоматических напоминаний;
- избыточность функционала для части пользователей.
Сервис предоставляет широкие возможности учета, однако требует больше времени на освоение.
## 2.4 Анализ веб-сервиса Smart Gardener
**Краткое описание:**  
Smart Gardener представляет собой инструмент для планирования выращивания растений и организации календаря работ.
**Функциональные возможности:**
- планирование мероприятий;
- формирование календаря;
- рекомендации по уходу;
- учет сезонных работ.
**Преимущества:**
- развитые инструменты планирования;
- структурированное представление данных;
- автоматизация расчётов сроков.
**Недостатки:**
- ориентированность на садовые культуры;
- отсутствие полноценного учета фактических действий;
- отсутствие русскоязычного интерфейса;
- ограниченная гибкость настроек.
Данный сервис демонстрирует сильную сторону в планировании, однако не охватывает задачи ведения истории ухода.
## 2.5 Выводы по результатам анализа
Проведённый анализ показал, что существующие веб-приложения решают отдельные задачи, связанные с уходом за растениями, однако не обеспечивают комплексного подхода.
Основные выявленные ограничения:
- недостаточная локализация; 
- раздельная реализация ключевых функций (планирование, учет, рекомендации);
- ограниченные возможности структурирования данных;
- различия в уровне сложности интерфейсов.  
Указанные особенности влияют на удобство работы с системой, скорость ввода информации и общее восприятие сервиса пользователем.

В связи с этим разрабатываемое веб-приложение должно быть ориентировано на:
- удобное представление информации;
- совмещение функций планирования и учета;
- доступность интерфейса;
- упрощение работы с данными.
