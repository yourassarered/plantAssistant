## Слайд 2. Актуальность темы

**Что говорить:**

Актуальность работы связана с тем, что при уходе за растениями информация часто хранится несистемно. Это затрудняет учет, усложняет контроль процедур ухода и делает неудобным анализ истории действий. Поэтому возникла необходимость разработки единого веб-приложения, объединяющего хранение данных, планирование ухода и удобное представление информации.

---

## Слайд 3. Цель, задачи и объект разработки

**Что говорить:**

Целью работы являлась разработка веб-приложения для систематизированного учета растений и планирования ухода. Для достижения цели были выполнены задачи анализа предметной области, выбора технологий, проектирования архитектуры системы, реализации функциональных модулей и проверки качества разработанного решения.

---

## Слайд 4. Основной функционал системы

**Что говорить:**

Приложение объединяет функции личного учета растений и социального взаимодействия пользователей. Пользователь может хранить информацию о растениях, настраивать параметры ухода, получать задачи, вести журнал выполненных действий и взаимодействовать с другими участниками через ленту, советы и подписки. Для администратора реализованы инструменты модерации и контроля контента.

---

## Слайд 5. Архитектура приложения

**Что говорить:**

Приложение построено по клиент-серверной архитектуре. Клиентская часть отвечает за интерфейс и взаимодействие с пользователем, а серверная часть выполняет обработку запросов, проверку прав доступа, работу с базой данных и возврат JSON-ответов. Такой подход позволяет независимо развивать frontend и backend части системы.

---

## Слайд 6. Backend-стек

**Что говорить:**

Серверная часть реализована на PHP 8.3 с использованием Laravel 13. Для хранения данных применяется PostgreSQL, а взаимодействие с базой данных организовано через Eloquent ORM. Авторизация пользователей реализована с помощью Laravel Sanctum. Для поддержки качества проекта используются OpenAPI или Swagger, PHPUnit и Laravel Pint.

---

## Слайд 7. Архитектура Laravel backend

**Что говорить:**

Laravel предоставляет слоистую структуру backend-приложения, включающую маршруты, контроллеры, request-классы, middleware, policies, сервисы и resources. Благодаря этому backend построен как поддерживаемая и масштабируемая система. Бизнес-логика отделена от работы с базой данных, а взаимодействие с таблицами реализовано через модели и Eloquent ORM. Frontend взаимодействует с сервером через единообразные JSON-ответы REST API.

---

## Слайд 8. Frontend-стек

**Что говорить:**

Frontend реализован как одностраничное приложение на Vue.js 3. Такой подход обеспечивает работу интерфейса без перезагрузки страниц и позволяет использовать реактивное состояние и переиспользуемые компоненты. Для маршрутизации применяется Vue Router, для управления состоянием — Pinia, а для работы с формами и валидацией используются VeeValidate и Zod.

---

## Слайд 9. Организация frontend-логики

**Что говорить:**

Клиентская логика разделена по функциональным областям. Компоненты отвечают за отображение интерфейса, Pinia-store управляют состоянием приложения, а единый API-клиент централизует взаимодействие с backend. Такой подход уменьшает дублирование логики и упрощает сопровождение проекта.

---

## Слайд 10. Docker и RoadRunner

**Что говорить:**

Docker используется для воспроизводимого развертывания приложения и уменьшения зависимости от локального окружения. Для повышения производительности применяется RoadRunner, который использует долгоживущие worker-процессы. Благодаря этому Laravel не инициализируется заново при каждом запросе, что снижает накладные расходы и ускоряет обработку API-запросов.

---

## Слайд 11. ER-диаграмма, СУБД и ORM

**Что говорить:**

В качестве системы управления базой данных используется PostgreSQL, поскольку приложение работает со связанной реляционной моделью данных. ER-диаграмма показывает основные сущности системы и связи между ними: пользователей, комнаты, растения, настройки ухода, журнал действий и социальные сущности. Для работы с базой данных используется Eloquent ORM, а структура базы данных управляется через миграции Laravel.

---

## Слайд 12. Безопасность и контроль качества

**Что говорить:**

Для защиты данных реализована токен-based авторизация через Laravel Sanctum, разграничение прав пользователей по ролям и контроль доступа через middleware и policies. Качество проекта поддерживается с помощью backend- и frontend-валидации, тестирования, документирования API и автоматической проверки стиля кода.

Реализованы backend и frontend части системы, социальные функции и административная панель. Архитектура проекта позволяет расширять функциональность и развивать систему в дальнейшем.
